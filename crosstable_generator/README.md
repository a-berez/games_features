# crosstable_generator

Скрипт строит «шахматку» попарного сравнения команд турнира по маскам взятий.

## Структура

```
crosstable_generator.py      # точка входа
crosstable/             # логика: API, матрица, экспорт
templates/crosstable.html
static/crosstable.css
static/crosstable.js
```

Результат собирается в каталог: по `-o` или `<ID>/` по умолчанию.  
При `--html` в каталог копируются `crosstable.css` и `crosstable.js`, создаются `index.html` и `<имя каталога>.data.json`.

## Зависимости

```bash
pip install requests openpyxl
```

## Использование

```bash
python crosstable_generator.py 12345 --html
python crosstable_generator.py 12345 --xlsx
python crosstable_generator.py 12345 --html --xlsx
python crosstable_generator.py 12345 -tc 15,15,15,15,15,15 --html -o chr2
python crosstable_generator.py 12345,67890 --html
python crosstable_generator.py 12345 -u mirror --html -v
```

Пример: `-o chr2 --html --xlsx` создаёт каталог `chr2/` с файлами `index.html`, `chr2.data.json`, `chr2.xlsx`, `crosstable.css`, `crosstable.js`. Ссылка на страницу: `/chr2/`.

### Аргументы

| Аргумент | Описание |
|----------|----------|
| `tournament_ids` | ID турнира или несколько ID через запятую |
| `-tc`, `--tour_comp` | Состав туров (число вопросов в каждом), например `15,15,15,15,15,15`. Без аргумента создаётся только лист `all` |
| `-o`, `--output` | Имя каталога для выходных файлов. По умолчанию `<ID>/`. При нескольких ID игнорируется |
| `-u`, `--url` | `base` (по умолчанию, `https://api.rating.chgk.net`) или `mirror` (`https://rating-api.pecheny.me`) |
| `--html` | Создать интерактивный HTML-файл |
| `--xlsx` | Создать XLSX-файл |
| `-v`, `--verbose` | Уровень логирования (`-v` — подробнее, `-vv` — отладка) |

Нужен хотя бы один из флагов `--html` или `--xlsx`.

## Результат

### XLSX (`--xlsx`)

Файл `<каталог>/<имя>.xlsx` с листом `all` и, при указании `--tour_comp`, листами `T1`, `T2`, …

### HTML (`--html`)

Файлы `<каталог>/index.html` и `<каталог>/<имя каталога>.data.json`. Данные подгружаются через `fetch` (нужен веб-сервер).

Интерактивная страница с:

- закреплёнными первыми двумя столбцами и строкой заголовков;
- вкладками «Все» / «Тур N»;
- выпадающим списком команд: полная матрица или сравнение выбранной команды со всеми соперниками;
- подсветкой чисел: зелёный • красный • синий • серый;
- названием турнира из API.

На каждом листе / в матрице:

- столбцы: `место`, `команда`, затем названия команд-соперников;
- в ячейке пересечения команд A и B: `X — Y — Z — W`, где
  - **X** — вопросы, взятые A, но не B;
  - **Y** — взятые B, но не A;
  - **Z** — взятые обеими;
  - **W** — не взятые обеими.
- снятые вопросы (`X` в маске) не учитаются;
- спорные без решения (`?`) считаются невзятыми;
- на диагонали — `×`.

Команды без маски пропускаются. Название берётся из `current.name`, при отсутствии — из `team.name`.
