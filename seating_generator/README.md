# Генератор рассадки команд

Скрипт, который пытается создать оптимальную рассадку команд с учётом ограничений.

## Описание

Это инструмент для организаторов игр в загадки, соревнований и других мероприятий, где требуется рассадить команды и/или индивидуальных участников в одном помещении таким образом, чтобы минимизировать нежелательное соседство.

Скрипт анализирует различные критерии (город, регион, номер команды и т. п.) и создаёт оптимальную рассадку, при которой команды с одинаковыми или похожими параметрами располагаются как можно дальше друг от друга.

### Основные возможности

- Загрузка списка команд с различными параметрами из Excel-файла
- Загрузка схемы рассадки с указанием доступных, недоступных и заранее занятых мест
- Настройка приоритета критериев для оптимальной рассадки
- Учёт диагонального соседства с настраиваемым весом штрафа
- Специальная обработка числовых параметров (например, номеров команд)
- Генерация нескольких вариантов рассадки с минимальными «штрафами»
- Сохранение результатов в Excel-файлы с итоговым штрафом на отдельном листе
- Неинтерактивный режим для пакетного запуска

## Требования

- Библиотеки:

  - openpyxl

## Использование

### Базовое использование

```
python seating_generator.py -i путь_к_файлу_команд.xlsx -s путь_к_схеме_рассадки.xlsx --no-confirm
```

Флаг `--no-confirm` пропускает интерактивный запрос порядка критериев. Без него скрипт покажет порядок столбцов и попросит подтвердить его.

Если запустить скрипт без параметров, он запросит пути к файлам интерактивно:

```
python seating_generator.py
```

### Параметры командной строки

#### Входные и выходные файлы

- `-i`, `--input` — путь к Excel-файлу со списком команд
- `-s`, `--seating` — путь к Excel-файлу со схемой рассадки
- `--output` — путь к выходному файлу лучшего варианта (по умолчанию `seating.xlsx` в папке скрипта). Альтернативные варианты сохраняются рядом с суффиксами `_1`, `_2` (например, `result_1.xlsx`)

#### Критерии рассадки

- `-o`, `--order` — порядок важности столбцов (например, `"3,1,2,4"`). При указании этого параметра подтверждение порядка не запрашивается
- `--no-confirm` — не запрашивать подтверждение порядка критериев
- `--ignore-number` — не учитывать столбец `number` при расчёте штрафов
- `-abc`, `--alphabetical_order` — устаревший алиас для `--ignore-number`
- `--number-close-diff` — максимальная разница номеров, при которой начисляется штраф за близость (по умолчанию: 3)
- `--diagonal-weight` — множитель штрафа для диагональных соседей (по умолчанию: 1.0; значение меньше 1 ослабляет штраф за диагональное соседство)

#### Параметры поиска

- `--initial-placements` — количество начальных вариантов рассадки (по умолчанию: 10)
- `--optimize-count` — сколько лучших начальных вариантов оптимизировать (по умолчанию: 3)
- `--max-iterations` — максимум итераций оптимизации на вариант (по умолчанию: 1000)

### Формат входных данных

#### Файл со списком команд (Excel)

Должен содержать следующие столбцы:

- `team` (обязательный) — название команды
- Другие столбцы — критерии для рассадки (город, регион, номер и т. д.). Столбец с номером всегда должен называться `number`, столбец с дополнительными параметрами, которые есть только у нескольких команд, — `extra`

Порядок столбцов слева направо соответствует важности критерия при рассадке (столбец `team` не участвует в расчёте штрафов). Например, если столбцы идут в порядке `number`, `team`, `region`, `town`, то при рассадке в первую очередь будут учитываться регионы, затем города, и только потом номера.

Специальные столбцы:

- `number` — если присутствует, обрабатывается особым образом: команды с близкими номерами получают более высокий штраф за соседство; для команд с малыми номерами штраф усиливается
- `extra` — если присутствует, команды с одинаковыми значениями в этом столбце не должны сидеть рядом

#### Файл со схемой рассадки (Excel)

Содержит схему помещения без заголовков, где:

- `X` — свободное место для размещения команды
- `O` — недоступное место (проходы, технические зоны и т. д.)
- **любой другой текст** — место, заранее закреплённое за командой с таким названием (команда должна быть во входном файле; при оптимизации не перемещается)

Пустые ячейки трактуются как `O`.

Пример схемы рассадки для помещения с 4 рядами по 4 стола:

|     | 1   | 2   | 3   | 4   |
|-----|-----|-----|-----|-----|
| **1** | X   | X   | X   | X   |
| **2** | X   | X   | X   | X   |
| **3** | X   | X   | X   | X   |
| **4** | X   | X   | X   | X   |

В Excel это выглядит как сетка 4×4 без заголовков — только символы `X` в каждой ячейке.

Пример с закреплённым местом и проходом:

|     | 1         | 2   | 3   | 4   |
|-----|-----------|-----|-----|-----|
| **1** | Команда А | X   | X   | O   |
| **2** | X         | X   | X   | X   |

Здесь «Команда А» закреплена на месте (1, 1), проход обозначен как `O`.

### Выходные данные

Скрипт генерирует до трёх файлов:

- `seating.xlsx` (или путь из `--output`) — лучший вариант рассадки
- `seating_1.xlsx`, `seating_2.xlsx` — альтернативные варианты (если `--optimize-count` больше 1)

Каждый файл содержит два листа:

- **Рассадка** — сетка с названиями команд; недоступные места помечены как `НЕДОСТУПНО`, незанятые свободные места — как `СВОБОДНО`
- **Информация** — итоговый штраф варианта

Лог работы записывается в файл `seating_generator.log` в папке скрипта.

## Алгоритм работы

1. **Загрузка данных**
   - Загрузка списка команд и их параметров
   - Загрузка схемы рассадки (включая закреплённые места)
   - Проверка достаточности свободных мест

2. **Определение порядка важности критериев**
   - Анализ столбцов во входном файле
   - Запрос подтверждения или изменения порядка у пользователя (если не указаны `--no-confirm` или `-o`)

3. **Генерация начальных вариантов рассадки**
   - Первый вариант строится жадным алгоритмом, остальные — случайно
   - Оценка качества каждого размещения (расчёт штрафов)
   - Выбор лучших вариантов для дальнейшей оптимизации

4. **Оптимизация рассадки**
   - Перебор обменов местами между подвижными командами
   - Оценка каждого обмена по системе штрафов
   - Сохранение обменов, улучшающих рассадку
   - Повторение до достижения локального оптимума или лимита итераций

5. **Сохранение результатов**
   - Сохранение лучшего варианта рассадки
   - Сохранение альтернативных вариантов (до трёх в общей сложности)

### Как считается штраф

- Соседними считаются места по сторонам и по диагонали (только клетки `X`)
- Каждая пара соседних команд учитывается один раз
- Для каждого критерия вес убывает с позицией в списке важности: `1`, `1/2`, `1/3`, …
- Совпадение значения критерия у соседей добавляет штраф
- Для столбца `number` штраф зависит от близости номеров (если не указан `--ignore-number)

## Примеры

В папке [examples](examples) находятся примеры файлов:

- `teams.xlsx` — список команд
- `places.xlsx` — схема рассадки (включая одно закреплённое место)
- `seating.xlsx`, `seating_1.xlsx`, `seating_2.xlsx` — примеры готовых результатов

### Пример запуска

```
python seating_generator.py -i examples/teams.xlsx -s examples/places.xlsx --no-confirm
```

### Пример с изменением порядка важности критериев

```
python seating_generator.py -i examples/teams.xlsx -s examples/places.xlsx -o "3,1,2,4"
```

Порядок `"3,1,2,4"` означает: сначала столбец 3 (`town`), затем 1 (`number`), затем 2 (`region`), затем 4 (`extra`).

### Пример с игнорированием номеров команд

```
python seating_generator.py -i examples/teams.xlsx -s examples/places.xlsx --ignore-number --no-confirm
```

### Пример с настройкой поиска и выходного файла

```
python seating_generator.py -i examples/teams.xlsx -s examples/places.xlsx --no-confirm --output result.xlsx --initial-placements 5 --max-iterations 200 --diagonal-weight 0.5
```

## Советы по использованию

- **Схема рассадки:** учитывайте не только физические ограничения помещения, но и желаемую плотность размещения команд. Подписи вроде «Вход» в ячейке схемы будут восприняты как закреплённая команда — используйте `O` для обозначений, не связанных с командами.

- **Номера команд:** если номера **не** отражают рейтинг или результаты предыдущих этапов, используйте `--ignore-number`.

- **Диагональное соседство:** по умолчанию штраф за диагональных соседей такой же, как за ортогональных. Чтобы ослабить его, укажите `--diagonal-weight 0.5` (или другое значение меньше 1).

- **Альтернативные варианты:** просмотрите все сгенерированные варианты рассадки — иногда второй или третий вариант может лучше соответствовать дополнительным критериям, не учтённым в скрипте.

- **Пакетный запуск:** для автоматизации всегда добавляйте `--no-confirm` или задавайте порядок критериев через `-o`.
